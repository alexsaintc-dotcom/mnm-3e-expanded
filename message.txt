  The Ghost Points Bug — Root Cause                                                                                                                                                 
                  
  I traced the full data flow through the upstream system's code. Here's what happens:                                                                                              
  
  How the upstream system handles extras/defauts on powers                                                                                                                          
                  
  The upstream system defines extras and defauts as ObjectField (plain objects, NOT arrays):                                                                                        
                  
  // pouvoir-data-model.mjs — upstream system                                                                                                                                       
  extras: new ObjectField(),   // expects: { "1": {name, data}, "2": {name, data} }                                                                                                 
  defauts: new ObjectField(),  // expects: { "1": {name, data}, "2": {name, data} }                                                                                                 
                                                                                                                                                                                    
  When a user drag-drops a modifier onto a power, the system does:                                                                                                                  
  // _onDrop in pouvoir-item-sheet.mjs — upstream                                                                                                                                   
  const dataMod = Object.keys(getData[`${getType}s`]);       // e.g. Object.keys(extras)                                                                                            
  const maxKeysMod = dataMod.length ? Math.max(...dataMod) : 0;                                                                                                                     
  update[`system.${getType}s.${maxKeysMod+1}`] = { name, data };                                                                                                                    
                                                                                                                                                                                    
  It uses numeric string keys ("1", "2", ...) on a plain object.                                                                                                                    
                                                                                                                                                                                    
  When a user deletes an extra, the system does:                                                                                                                                    
  this.item.update({[`system.extras.-=${id}`]: null});  // Foundry object key deletion                                                                                              
                                                                                                                                                                                    
  The .-= syntax is Foundry's way to delete a key from an object. It does NOT work correctly on arrays.                                                                             
                                                                                                                                                                                    
  Your build script outputs arrays                                                                                                                                                  
                                                                                                                                                                                    
  In build.js lines 97 and 115:                                                                                                                                                     
  const extrasArray = [];  // ← ARRAY
  // ...                                                                                                                                                                            
  extrasArray.push({ name, data: { description, cout } });                                                                                                                          
  // ...

  ★ Insight ─────────────────────────────────────                                                                                                                                   
  Array vs Object in Foundry's -= deletion — When Foundry processes system.extras.-=1, it removes key "1" from an object. On a plain object {1: {...}}, this works perfectly. On an 
  array, JavaScript's delete arr[1] creates a sparse array — the slot becomes undefined but the array length stays the same. When Foundry serializes and reloads this sparse array, 
  the undefined slot can become null, which then crashes or misbehaves in the extraCost getter (which tries to access .data.cout on each entry). The leftover entries keep their    
  cost contribution alive — ghost points.                                                                                                                                           
  ─────────────────────────────────────────────────
                                                                                                                                                                                    
  There's also a second bug: special field value                                                                                                                                    
                                                                                                                                                                                    
  In build.js:136:                                                                                                                                                                  
  let specialToggle = "simple"; // Default to Standard
                                                                                                                                                                                    
  But the upstream system expects "standard", not "simple". The cost calculation checks:
